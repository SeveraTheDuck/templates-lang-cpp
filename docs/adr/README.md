# Architecture Decision Records

An ADR is a short document capturing an important architectural decision:
what was decided, why, and what alternatives were considered.

## When to write an ADR

- Choosing a technology, library, or tool
- Defining a structural pattern or convention
- Making a decision with non-obvious trade-offs
- Reversing or superseding a previous decision

## How to write an ADR

Copy `0000-template.md`, increment the number, and fill in the sections.
Commit it alongside the code change it describes.

## Status values

- **proposed** — under discussion, not yet adopted
- **accepted** — adopted and in effect
- **superseded by [NNNN](NNNN-title.md)** — replaced by a later decision
- **deprecated** — no longer relevant
